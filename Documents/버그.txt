 - UI에 있는 이미지가 떨리는 버그
    ㄴ 캔버스 렌더 모드가 카메라로 되어 있기 때문으로 예상
    ㄴ 추후 오버레이로 바꿔주면 해결 됨.
    ㄴ 작업상 카메라가 편해서 나중에 수정할 예정.
    ㄴ 캐릭터 움직임을 LateUpdate로 수정했더니 UI 뿐만 아니라 플레이어도 이동할 때 떨리게 됨.
    ㄴ RigidBody가 없는 고정 충돌체(아마 isTrigger 체크되어 있는 Collider)를 움직이면 물리 월드 재구성으로 오버헤드가 발생한다고 함. 플레이어와 적, 투사체 와 같은 위에 해당하는 오브젝트들에게 RigidBody 넣어줘서 해결.
    ㄴ 참고: https://202psj.tistory.com/1263

 - 버프 합성 UI 에 양쪽에 같은 버프를 넣었을 때, 두 번째 버프를 눌러도 첫 번째 버프가 먼저 빠지는 버그
    ㄴ 현재 로직을 첫 번째 체크 후 두 번째를 체크하기 때문.

 - 최단 경로가 아니고 돌아서 가던 버그
    ㄴ 내가 만든 우선순위 큐가 우선순위가 같은 경우 먼저 푸쉬된 것을 먼저 팝하는데 반복문에서 주변 방향 체크할 때 좌우보다 위 방향을 먼저 체크하기 때문에 발생. 방향 우선순위를 { 아래, 오른쪽, 왼쪽, 위쪽, 우하단, 좌하단, 우상단, 좌상단 } 으로 수정해서 해결

 - 적 소환할 때 가끔씩 NullReferenceException 으로 오류 뜨던 버그
    ㄴ 디스폰할 때 m_State(상태)를 Idle로 바꿔주지 않아서 생김. (Finallize 문제)

 - 적 생성 시스템이 방에서 나갈 때 적을 제대로 지우지 않고, 방 클리어도 제대로 확인하지 못하는 버그
    ㄴ ObjectPoolItemBase의 Initialize와 Finallize에서 onSpawn, onDespawn 이벤트 호출하는 부분을 외부로 빼려고 제거해서 적을 잡았을 때 이벤트가 호출되지 않았음

 - 버프 보상 UI 창에서 선택한 버프와 실제로 획득한 버프가 다른 버그
    ㄴ 반복문 밖에서 생성한 변수 재사용해서 이벤트 호출이 뭔가 꼬인듯한 느낌
    ㄴ 반복문 안에서 매번 변수 다시 생성하도록해서 해결
